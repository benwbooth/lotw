# Verdict: the table(s) at PRG 0x14000 (bank 10) are SOUND DATA, not metasprites

**Charter:** resolve the contested identity of the table(s) at PRG file offset `0x14000`
(bank `0x0A`). Recon agent 06 (audio) read them as the sound driver's song-header +
SFX pointer tables; agent 07 (graphics) read the *same bytes* as a metasprite
animation/frame-pointer system (`06 NN 86` records). Both cannot be right.

## VERDICT (HIGH confidence)

**Agent 06 is correct. Agent 07's metasprite reading of `0x14000` is wrong.** The region at
PRG `0x14000` is the **sound engine's data block for bank `0x0A`**:

- `0x14000` = **song-header pointer table** (`L_FC08` reads `$8000,X`).
- `0x14014` = **SFX pointer table** (`L_FA60` reads `$8014,X`).
- `0x14062+` = the **8-byte-per-channel song header descriptors**.
- `0x141A2+` = **note-event streams** (music + SFX), `[dur][note]` grammar with `$FF cc aa`.

The bank-`0x0A`/`0x0B` pair is **only ever paged into the `$8000`/`$A000` window by the sound
driver**, and the **only code that indexes that window after the swap is the sound driver**.
The bank-13 OAM/sprite engine (PRG `~0x1A400`) never reads this region; its `$Bxxx`/`$B3xx`
"frame data" was an offset miscalculation (8 KiB vs 16 KiB MMC3 windows) plus a zero-page
pointer mistaken for a ROM pointer.

---

## Evidence chain

### 1. Bank `0x0A` is selected ONLY by the sound driver

Exhaustive scan of all code (`disasm/bankfix.s` = fixed banks 14/15; `disasm/bank13.s`): the
only immediates `#$0A`/`#$0B` that reach an MMC3 bank register are in two sound routines.
Every other `#$0A`/`#$0B` is `CMP`/`SBC`/`ADC`/`ASL` arithmetic on game values, never a
`$8000`/`$8001` store.

`L_FD74` (`$FD74`, `disasm/bankfix.s:7931`) — default music banks:
```
LDX #$06 ; LDY #$0A ; STX $8000 ; STY $8001   ; R6 = $0A  -> $8000
INX      ; INY      ; STX $8000 ; STY $8001   ; R7 = $0B  -> $A000
```

`L_FD87` (`$FD87`, `:7941`) — per-song banks from the `$34/$35` shadow pair:
```
LDA #$06 ; STA $8000 ; LDA $34 ; STA $8001     ; R6 = $34
LDA #$07 ; STA $8000 ; LDA $35 ; STA $8001     ; R7 = $35
```

The `$34/$35` music-bank shadow is **written only in `L_FC08`** (`:7720-7722`,
`STX $34 / INX / STX $35`, with `X = $0A` for songs <10 or `$0C` for songs >=10) and **read
only in `L_FD87`**. No other code touches `$34/$35`. The OAM/sprite engine cannot set its own
bank anyway — bank 13 *lives in* the swapped window.

### 2. The consumer of `0x14000` is the song loader `L_FC08`

`L_FC08` (`$FC08`, `disasm/bankfix.s:7713`), called from 4 sites
(`:2260,:3691,:3697,:4555`):
```
LDX #$0A ; LDA $8E ; CMP #$0A ; BCC + ; LDX #$0C   ; pick music bank by song id $8E
STX $34 ; INX ; STX $35 ; JSR L_FD87               ; map bank 0A/0B (or 0C/0D) to $8000/$A000
LDA $8E ; CMP #$0A ; BCC + ; SBC #$0A               ; song index within bank
ASL A ; TAX
LDA $8000,X ; STA $0E ; LDA $8001,X ; STA $0F       ; read song-header pointer from 0x14000 table
; then copy 4 channels x 8 bytes from (header) into channel state at $93 (LDY #$07 loop)
```
This reads the `$8000`-table at PRG `0x14000` exactly as agent 06 described and copies four
8-byte channel descriptors into the channel state at `$93/$A3/$B3/$C3`.

### 3. The consumer of `0x14014` is the SFX engine `L_FA60`

`L_FA60` (`$FA60`, `disasm/bankfix.s:7483`), called once per frame from the `$F89A` tick:
```
LDA $8F ; ... (priority compare $90/$91) ...
LDA $8F ; ASL A ; TAX
LDA $8014,X ; STA $D5 ; LDA $8015,X ; STA $D6        ; read SFX-stream pointer from 0x14014 table
LDA #$80 ; STA $D4                                   ; mark SFX active
```
`$8F` = requested SFX id; `*2` indexes the SFX pointer table at PRG `0x14014`. Matches the
ascending `$B3AB..$B9E9` pointer block at `0x14014`.

### 4. The bytes themselves decode as sound, under the CORRECT 8 KiB banking

MMC3 windows are **8 KiB**: with R6=`$0A` the `$8000-$9FFF` window is bank `0x0A`
(file `0x14000`), and the `$A000-$BFFF` window is bank `0x0B` (file `0x16000`). So a pointer
like `$B3AB` resolves to **file `0x173AB`** (`0x16000 + (0xB3AB-0xA000)`), NOT `0x153AB`.

- Song 0 ch0 stream pointer `$81A2` -> PRG `0x141A2`:
  `FF 00 0B  FF 01 FF  48 10  48 20  0C 28 ...`
  = cmd0(instr 0x0B), cmd1(tempo), then `[dur][note]` pairs. Clean music stream.
- SFX index 1 pointer `$B3AB` -> **file `0x173AB`** (bank `0x0B`, `$A000` window):
  `FF 00 24  FF 01 00  01 56  01 4C  01 60  01 65 00 ...`
  = cmd0(instr 0x24), cmd1(tempo), short `[dur][note]` pairs ending `00`. Clean SFX stream.

Both decode as the documented `[dur][note]` / `$FF cc aa` grammar — they are sound streams.

### 5. Where agent 07's metasprite reading went wrong

- **8 KiB-vs-16 KiB window error.** Agent 07 computed `$B3AB -> 0x153AB` by treating bank
  `0x0A` as if it filled the whole `$8000-$BFFF` (16 KiB). It does not; `$Bxxx` is the `$A000`
  window (bank `0x0B`). The real SFX data is at `0x173AB`, where it decodes as sound. The
  `06 NN 86` bytes at `0x153AB` are unrelated interior bytes of bank `0x0A`'s data
  (around the song streams near `$93xx`), never pointed to by any frame table.

- **The real OAM engine does not touch this region.** PRG `~0x1A400` (bank 13) is genuinely
  the sprite/OAM engine, but it indexes RAM/OAM, not bank `0x0A` ROM:
  - writes OAM page `$0400` (`BD 01 04` `LDA $0401,X`, `9D 01 04` `STA $0401,X`) and a
    `$0180` shadow buffer (`BD 80 01`).
  - uses zero-page pointer `$7A/$7B`, where `$7B` is explicitly set to `$00`
    (`A9 00 85 7B` at PRG `0x1A344`) and `$7A` to `$B2/$B3/$B4`
    (`A9 B3 85 7A`, `A9 B4 85 7A`, `A9 B2 85 7A` near PRG `0x1A4A0-0x1A4F0`). That makes the
    pointer **`$00B2-$00B4` in zero page / RAM**, a sprite-state structure — NOT a ROM
    `$B3xx` address into bank `0x0A`. There is no `LDA ($7A),Y` reaching ROM here.
  The `$B3`-ish constants agent 07 saw are RAM pointer high-bytes, coincidentally similar to
  the SFX pointer values, which led to the conflation.

## Proposed data-label symbols (bank `0x0A`, mapped at `$8000`/`$A000`)

| Symbol | CPU addr | PRG / file | Kind | Meaning |
|---|---|---|---|---|
| `SND_SONG_PTR_TABLE` | `$8000` | `0x14000` / `0x14010` | data (LE word array) | song-header pointers, indexed by `(song_id [<10]) * 2`; read by `L_FC08` |
| `SND_SFX_PTR_TABLE`  | `$8014` | `0x14014` / `0x14024` | data (LE word array) | SFX-stream pointers, indexed by `sfx_id * 2`; read by `L_FA60` |
| `SND_SONG_HDR_BASE`  | `$8062` | `0x14062` / `0x14072` | data | first song header; 4 channel descriptors x 8 bytes each |
| `SND_STREAM_DATA_0A` | `$81A2..` | `0x141A2..` | data | music/SFX event streams in bank `0x0A` (`$8000` window) |
| `SND_STREAM_DATA_0B` | `$B3AB..` | `0x173AB..` (bank `0x0B`) | data | SFX/continuation event streams in bank `0x0B` (`$A000` window) |

Song-header descriptor (8 bytes, copied verbatim to channel state by `L_FC08`):
`[+0,+1]` = silence-stub ptr (`$8004`); `[+2,+3]` = current stream ptr; `[+4,+5]` = stream
start/loop ptr; `[+6,+7]` = init params (duration / flags-transpose).

## Net effect on the recon SUMMARY

- SUMMARY section 3, open question #1 (`0x14000` identity conflict) is **RESOLVED in favor of
  the audio reading (agent 06)**. Agent 07's section 4 ("Metasprite / OAM assembly tables —
  PRG `0x14000` region") should be retracted; the real OAM engine at PRG `0x1A400` consumes
  RAM/OAM, not bank-`0x0A` ROM.
- SUMMARY open question #2 (per-bank `$8000`/`$A000` window assignment): for the music data,
  **bank `0x0A` -> `$8000`, bank `0x0B` -> `$A000`** (set by `L_FD74`/`L_FD87`). `$Bxxx`
  pointers in the bank-`0x0A` tables therefore live in bank `0x0B` at file `0x16000+`.
